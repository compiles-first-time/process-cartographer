# Critic checklist — Internationalization (i18n) + Localization (l10n)

> Used by the Critic subagent when reviewing discovery artifacts. Per [ADR-0026](../../../adr/0026-discovery-gate.md).

## Scope decision

- [ ] Decision recorded: i18n at launch / i18n later / never
- [ ] If "later", record what would force i18n now (regulatory, market entry, etc.)

## Locale infrastructure (if i18n is in scope)

- [ ] Locale identifier strategy (BCP 47 tags)
- [ ] Default locale + fallback chain
- [ ] String externalization (no hardcoded user-visible strings in code)
- [ ] Translation workflow (vendor / in-house / community)

## Locale-sensitive operations

- [ ] Date formatting (Intl.DateTimeFormat)
- [ ] Number formatting (Intl.NumberFormat)
- [ ] Currency formatting + currency code per locale
- [ ] Plural rules (Intl.PluralRules; ICU MessageFormat)
- [ ] Collation (Intl.Collator) for sorting

## RTL support

- [ ] Decision recorded: RTL supported / not yet
- [ ] If supported: dir attribute, logical CSS properties (inline-start vs left)

## Content + data

- [ ] User content allows Unicode (UTF-8 end-to-end)
- [ ] Database collation + indexes Unicode-aware
- [ ] Search / sort honors locale

## Process

- [ ] Pseudo-localization in test envs (e.g., wrap strings with [!!]) to surface gaps
- [ ] Translator notes captured for ambiguous strings
