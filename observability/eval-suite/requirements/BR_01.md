## Known limitations (tracked, not hidden)

- **Substring matching (BR-01_SE-04):** the guard classifies command *text*, so a command that merely quotes a destructive pattern is asked-about. `ask` (not `deny`) bounds the cost. A shell-AST parse would remove the false positive â€” future work.
- **Contained scope = worktrees only (v1):** scratchpad containment is a documented v1.1 extension; scratchpad destructive ops currently `ask` (conservative).
- **Contained-scope is a best-effort `ask`â†’`allow` downgrade, never a `deny`â†’`allow` bypass (critic C1, 2026-07-07):** only a *single, unchained, un-substituted, un-commented* command is trusted. A compound/commented command that merely mentions `.worktrees/` (e.g. `rm -rf /data && cd .worktrees/x`, or `rm -rf /data #.worktrees`) falls through to `ask`. Regression-locked by `CF-BYPASS-01`. Full shell-AST target extraction remains future work; the deny tier is unaffected.
- **`ask` in headless runs:** resolves to the harness default (typically deny) â€” the safe failure direction.
